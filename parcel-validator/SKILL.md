# Parcel Validator - Property Data Cross-Reference

## Purpose

Compare property data from three independent sources — **King County Assessor** (via parcel-radar), **Zillow** (via zillow-scraper), and **King County iMap** (via kingcounty-imap) — to highlight discrepancies and give a comprehensive view of a property. Output a **PDF report** with embedded screenshots.

## Workflow

### Step 1: Collect Data from All Sources (Parallel)

Use the `Agent` tool to dispatch **three parallel subagents**:

**Agent A — Parcel Radar (King County Assessor):**
- Invoke the `parcel-radar` skill for the given address
- Extract and return these fields in structured JSON:
  - Square footage (building)
  - Lot size (sq ft and/or acres)
  - Year built
  - Bedrooms
  - Bathrooms
  - Assessed value (land, improvement, total)
  - Property type / use
  - Stories / floors
  - Condition grade
  - Zoning
- Also extract the **latitude and longitude** from the geocode response (needed by Agent C)

**Agent B — Zillow Scraper:**
- Invoke the `zillow-scraper` skill for the same address
- Extract and return these fields in structured JSON:
  - Square footage (listing)
  - Lot size
  - Year built
  - Bedrooms
  - Bathrooms
  - List price / Zestimate
  - Price per sq ft
  - Property type
  - Days on market
  - HOA fees
  - Parking / garage
  - Heating / cooling
  - Tax assessed value (from Zillow)
  - Last sold date and price
- **IMPORTANT:** Save a screenshot as `zillow_property.png` in the working directory

**Agent C — King County iMap:**
- Invoke the `kingcounty-imap` skill for the same address
- Use coordinates from parcel-radar geocode if available, otherwise let iMap geocode
- Run the imap_scrape.py script:
  ```bash
  python3 ~/.claude/skills/kingcounty-imap/imap_scrape.py \
      "<address>" imap_<short_address>.png \
      --lat <lat> --lon <lon>
  ```
- This produces two screenshots:
  - `imap_<short_address>.png` — clean map with environmental/hazard layers
  - `imap_<short_address>_with_layers.png` — map with Layer List panel visible

### Step 2: Normalize and Compare

Once all agents return, normalize the data and build a comparison:
- Convert all areas to sq ft for comparison
- Round to reasonable precision (e.g., sq ft to nearest integer)
- Treat null/missing as "N/A" — do not treat missing as zero

### Step 4: Convert to PDF

Use **WeasyPrint** to convert the markdown report to PDF. WeasyPrint renders HTML, so first convert MD → HTML → PDF.

Run this Python script via Bash:

```python
python3 -c "
import markdown
from weasyprint import HTML
import sys, os

md_file = sys.argv[1]
pdf_file = md_file.replace('.md', '.pdf')
base_dir = os.path.dirname(os.path.abspath(md_file)) or '.'

with open(md_file, 'r') as f:
    md_content = f.read()

html_body = markdown.markdown(md_content, extensions=['tables', 'fenced_code'])

html_full = f'''<!DOCTYPE html>
<html>
<head>
<meta charset=\"utf-8\">
<style>
  @page {{ size: A4; margin: 2cm; }}
  body {{ font-family: -apple-system, 'Helvetica Neue', Arial, sans-serif; font-size: 11pt; line-height: 1.6; color: #1a1a1a; }}
  h1 {{ font-size: 20pt; border-bottom: 2px solid #2563eb; padding-bottom: 8px; color: #1e3a5f; }}
  h2 {{ font-size: 15pt; color: #2563eb; margin-top: 24px; border-bottom: 1px solid #e5e7eb; padding-bottom: 4px; }}
  h3 {{ font-size: 12pt; color: #374151; }}
  table {{ border-collapse: collapse; width: 100%; margin: 12px 0; font-size: 10pt; }}
  th, td {{ border: 1px solid #d1d5db; padding: 6px 10px; text-align: left; }}
  th {{ background-color: #f0f4ff; font-weight: 600; color: #1e3a5f; }}
  tr:nth-child(even) {{ background-color: #f9fafb; }}
  img {{ max-width: 100%; height: auto; margin: 12px 0; border: 1px solid #e5e7eb; border-radius: 4px; }}
  strong {{ color: #1e3a5f; }}
  hr {{ border: none; border-top: 1px solid #e5e7eb; margin: 20px 0; }}
  p {{ margin: 6px 0; }}
  em {{ color: #6b7280; font-size: 9pt; }}
</style>
</head>
<body>
{{html_body}}
</body>
</html>'''

HTML(string=html_full, base_url=base_dir).write_pdf(pdf_file)
print(f'PDF saved: {{pdf_file}}')
" "<md_file_path>"
```

**IMPORTANT:** The `base_url` parameter must point to the directory containing the images so that `![](image.png)` references resolve correctly.

- [ ] Square footage (building/living area)
- [ ] Lot size

- [ ] Stories / floors
- [ ] Property type

- [ ] Zoning

- [ ] Days on market

- [ ] School district

## Error Handling

- If parcel-radar fails (address not in King County): Report that county data is unavailable and present Zillow data only with a note
- If zillow-scraper fails (blocked, no zpid found): Report that Zillow data is unavailable and present county data only
- If kingcounty-imap fails: Skip the iMap screenshots section in the report, note "iMap screenshots unavailable"
- If both data sources fail: Report the failure and suggest the user try again or verify the address

## Notes

- This skill is designed for **King County, WA** properties (parcel-radar limitation)
- County Assessor data reflects the most recent assessment, which may lag behind renovations
- Zillow data may reflect seller-reported updates that haven't been officially recorded
- Discrepancies don't necessarily mean one source is wrong — they often reflect different measurement standards or timing
- PDF conversion requires `weasyprint` and `markdown` Python packages (`pip3 install weasyprint markdown`)
- All images must be in the same directory as the .md file for PDF embedding to work
